PROJECT OVERVIEW
================================================================================

I want to build a "Visual Prompter" - a web-based application that allows users 
to CREATE AI PROMPTS VISUALLY through 3D diagrams, flowcharts, and connected 
shapes, similar to how designers work in Fusion 360, SolidWorks, or Visio.

The core concept is: DRAW YOUR PROMPT, DON'T WRITE IT.

This tool is designed for people who:
- Know how to design/draw but don't know how to program
- Are visual thinkers who struggle to express technical requirements in text
- Want to create complex, structured AI prompts without writing paragraphs
- Need to design software architectures visually before generating prompts

================================================================================
CORE CONCEPT
================================================================================

Instead of writing: 
"Create a PHP application with a MySQL database that has a users table 
connected to an orders table with a one-to-many relationship..."

The user DRAWS:
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│  📦 MySQL   │────────▶│  📋 Users   │────────▶│  📋 Orders  │
│  Database   │         │   Table     │  1:N    │   Table     │
│             │         │             │         │             │
│ Host: local │         │ - id        │         │ - id        │
│ Port: 3306  │         │ - name      │         │ - user_id   │
└─────────────┘         │ - email     │         │ - total     │
                        └─────────────┘         └─────────────┘
                                │
                                ▼
                        ┌─────────────┐
                        │  🔷 PHP     │
                        │  Backend    │
                        │             │
                        │ Framework:  │
                        │ Laravel     │
                        └─────────────┘

Then clicks "GENERATE PROMPT" and the system converts this visual diagram 
into a detailed, well-structured AI prompt that can be copied and used with 
ChatGPT, Claude, or any AI assistant.

DETAILED REQUIREMENTS
================================================================================

1. CANVAS & WORKSPACE
─────────────────────
- 3D canvas using Three.js (already implemented)
- Infinite workspace with pan, zoom, and rotate
- Grid background for alignment
- Dark theme with modern UI (already styled)
- Save/Load projects as JSON files

2. SHAPE LIBRARY (Nodes/Boxes)
──────────────────────────────
The system should support various shapes, each representing a different 
concept in software development:

SHAPE               │ MEANING                    │ COLOR
────────────────────┼────────────────────────────┼──────────────
□ Rectangle/Box     │ Component/Module/File      │ Blue (#6366f1)
○ Circle            │ Process/Function           │ Green (#10b981)
◇ Diamond           │ Condition/Decision         │ Yellow (#f59e0b)
▭ Rounded Rectangle │ User Interface/Page        │ Purple (#8b5cf6)
⬡ Hexagon           │ API Endpoint               │ Cyan (#06b6d4)
⊡ Cylinder          │ Database/Storage           │ Orange (#f97316)
△ Triangle          │ Start/End Point            │ Red (#ef4444)
⬢ Octagon           │ External Service           │ Pink (#ec4899)

3. NODE PROPERTIES (Editable Fields)
────────────────────────────────────
Each shape/node should have editable properties:

GENERAL PROPERTIES:
- Title/Name (required)
- Description (optional)
- Type (from predefined list)
- Tags/Labels

TECHNICAL PROPERTIES (context-dependent):
For Database nodes:
  - Database Type (MySQL, PostgreSQL, MongoDB, etc.)
  - Host, Port, Username
  - Tables (sub-list with columns)
  
For Backend nodes:
  - Language (PHP, Python, Node.js, etc.)
  - Framework (Laravel, Django, Express, etc.)
  - Endpoints list
  
For Frontend nodes:
  - Framework (React, Vue, Angular, HTML)
  - Pages list
  - Components list

For API nodes:
  - Method (GET, POST, PUT, DELETE)
  - Endpoint URL
  - Request/Response structure
  - Authentication type

For File nodes:
  - Filename
  - File type
  - Purpose/Function

4. CONNECTIONS (Lines/Relationships)
────────────────────────────────────
Users should be able to draw lines between nodes to show relationships:

LINE TYPE           │ MEANING                    │ STYLE
────────────────────┼────────────────────────────┼──────────────
─────────────────▶  │ Data Flow                  │ Solid + Arrow
─ ─ ─ ─ ─ ─ ─ ─ ▶  │ Optional/Conditional       │ Dashed + Arrow
════════════════▶   │ Strong Dependency          │ Double line
─────────○─────     │ One-to-One                 │ Circle end
─────────<─────     │ One-to-Many                │ Crow's foot
─────────◇─────     │ Many-to-Many               │ Diamond end

Connection labels:
- Relationship type (1:1, 1:N, N:N)
- Data being transferred
- Condition/trigger

5. LAYERS & GROUPING
────────────────────
- Group related nodes together
- Create layers for different aspects:
  - Database Layer
  - Backend Layer  
  - Frontend Layer
  - API Layer
  - External Services Layer
- Collapse/expand groups
- Color-code by layer

6. PROMPT GENERATION ENGINE
───────────────────────────
The heart of the system - converts visual diagram to text prompt:

INPUT: JSON structure of nodes and connections
OUTPUT: Well-formatted AI prompt

The generated prompt should include:
a) PROJECT OVERVIEW
   - Derived from the overall structure
   - Main components identified
   
b) TECHNICAL STACK
   - All technologies mentioned in nodes
   - Versions if specified
   
c) DATABASE STRUCTURE
   - Tables, columns, relationships
   - Extracted from database/table nodes
   
d) API ENDPOINTS
   - All routes with methods
   - Request/response structures
   
e) FILE STRUCTURE
   - Folder hierarchy
   - File purposes
   
f) RELATIONSHIPS & DATA FLOW
   - How components connect
   - Data transformation points
   
g) SPECIAL REQUIREMENTS
   - From notes/annotations
   - Constraints and rules

7. EXPORT OPTIONS
─────────────────
- Generate Prompt → Copy to clipboard / Download as .txt
- Export Diagram → PNG, SVG, PDF
- Export Project → JSON (for later editing)
- Share Link → Generate shareable URL
