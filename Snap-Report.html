What is WINGET?

WINGET (Windows Package Manager) is Microsoft's official CLI tool Command Line Interface - A text-based way to interact with your computer for installing, updating, and managing software packages on Windows 10/11. Think of it as an "app store" you control from the command line! 🚀

⚡

Lightning Fast

Install apps in seconds with a single command. No clicking through installers!

🔄

Easy Updates

Update all your apps at once with `winget upgrade --all`

Search Anything

Find software from Microsoft's massive repository of packages

Prerequisites Check

Before installing Node.js, make sure you have:

✓ Windows 10 (1809+) or Windows 11
✓ WINGET installed (comes pre-installed on Windows 11)
✓ PowerShell or Command Prompt (Run as Administrator recommended)
✓ Internet connection to download packages
PS C:\> winget --version

Run this to check if WINGET is installed. You should see a version number like "v1.x.x"

Install Node.js - The Main Event!

1

Open PowerShell as Administrator

Right-click the Start button → Select "Windows Terminal (Admin)" or "PowerShell (Admin)"

2

Run the Installation Command

Copy and paste the command below:

PS C:\> winget install OpenJS.NodeJS.LTS
0
🎉 People installed Node.js today using WINGET!
3

Wait for Installation

WINGET will download and install Node.js automatically. This usually takes 1-2 minutes.

4

Verify Installation

Close and reopen your terminal, then run:

PS C:\> node --version && npm --version

All Node.js WINGET Commands

🆕 Latest LTS (Recommended)

winget install OpenJS.NodeJS.LTS

⚡ Latest Current Version

winget install OpenJS.NodeJS

🔄 Update Node.js

winget upgrade OpenJS.NodeJS.LTS

🗑️ Uninstall Node.js

winget uninstall OpenJS.NodeJS.LTS

Troubleshooting Common Issues

❓

"winget" is not recognized

Install App Installer from Microsoft Store, or update Windows to the latest version.

❓

"node" not found after install

Close ALL terminal windows and open a new one. The PATH needs to refresh!

❓

Permission denied errors

Make sure you're running PowerShell as Administrator (right-click → Run as Admin)
